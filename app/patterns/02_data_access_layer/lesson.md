# 02. Data Access Layer — 학습 가이드

## 의도

데이터 접근의 **유일한 통로**를 만든다. 페이지/액션은 직접 DB에 접근하지 않고 DAL 함수만 호출.

## 한 곳에 모이는 것

1. **권한 검증** — 누가 어떤 데이터를 볼 수 있나
2. **캐시 키 설계** — 사용자/언어/테넌트 별 분리
3. **데이터 정제** — 화면이 정말 필요한 필드만 노출 (over-fetching 방지)
4. **에러 처리** — 일관된 도메인 에러 형식

## server-only 마커

```ts
import "server-only";
```

이 import 가 있는 모듈은 클라이언트 번들로 들어가는 순간 빌드 실패. 비밀 키 / DB 코드 / 권한 로직이 새는 것을 방지하는 안전망.

## 디렉토리 구조 (제안)

```
lib/
├── notes-dal.ts
├── users-dal.ts
├── auth.ts        ← 현재 사용자 / 권한
└── db.ts          ← Prisma/Drizzle 클라이언트 (단일 인스턴스)
```
